# 使用说明

## 快速开始

### 1. 安装依赖
```bash
npm install
```

### 2. 初始化数据库
在 Supabase 或你的 PostgreSQL 数据库中执行 `init-db.sql` 文件中的 SQL 语句来创建必要的表结构。

你可以通过以下方式执行：
- 登录 Supabase Dashboard，进入 SQL Editor，粘贴并执行 `init-db.sql` 中的内容
- 或使用 psql 命令行工具连接数据库后执行

### 3. 配置环境变量
`.env` 文件已经配置好数据库密码，你可以根据需要修改 `SESSION_SECRET`（建议修改为一个随机字符串以提高安全性）。

### 4. 启动应用
```bash
npm start
```

或使用开发模式（自动重启）：
```bash
npm run dev
```

### 5. 访问应用
打开浏览器访问：http://localhost:3000

## 首次使用

1. 点击"注册新账号"按钮
2. 输入用户名和密码（密码至少6位）
3. 注册成功后会自动登录

## 功能说明

### 每日打卡
- 每天只能打卡一次
- 上传当天的脸部照片（支持 JPG/JPEG/PNG 格式，最大 5MB）
- 系统会自动计算连续打卡天数
- 如果中断打卡，连续天数会重新从1开始计算

### 统计信息
- **当前连续天数**：今天打卡后的连续天数
- **最长连续天数**：历史最长的连续打卡记录
- **总打卡次数**：累计打卡的总次数

### 四种视图模式

#### 1. 网格视图
- 以网格形式展示所有打卡照片
- 鼠标悬停可放大查看
- 显示日期和打卡天数

#### 2. 时间线视图
- 按时间顺序纵向排列
- 清晰展示每次打卡的时间线
- 适合查看整体进度

#### 3. 对比视图
- 点击任意两张照片进行对比
- 可以直观看到脸部变化
- 点击"清除选择"重新选择照片

#### 4. 日历视图
- 以日历形式展示打卡记录
- 可以切换月份查看历史记录
- 打卡日期会显示照片缩略图和天数徽章

### 自动登录功能
- 勾选"记住我"后，30天内无需重复登录
- 不勾选则保持登录1天
- 可随时点击"登出"退出登录

## 注意事项

1. **隐私保护**：只有登录用户才能访问网站内容
2. **照片存储**：所有照片以 Base64 编码字符串形式存储在数据库中，不需要额外的文件存储空间
3. **数据安全**：密码使用 bcrypt 加密存储
4. **连续打卡**：必须每天都打卡才能保持连续天数，中断后会重新计算
5. **数据库大小**：由于照片存储在数据库中，建议定期检查数据库大小

## 技术栈

- **后端**：Node.js + Express
- **数据库**：PostgreSQL (Supabase)
- **前端**：原生 HTML/CSS/JavaScript
- **认证**：Session + Cookie
- **文件上传**：Multer

## 故障排除

### 无法连接数据库
- 检查 `.env` 文件中的 `DATA_BASE_PASSWORD` 是否正确
- 确认网络连接正常
- 验证数据库表是否已创建

### 照片上传失败
- 确认照片格式为 JPG/JPEG/PNG
- 检查照片大小是否超过 5MB
- 确认数据库连接正常
- 检查数据库是否有足够的存储空间

### 无法登录
- 确认用户名和密码是否正确
- 检查数据库连接是否正常
- 查看浏览器控制台是否有错误信息
